Se vuoi usare MAMP su altre macchine:

MAMP/conf/php8.0.1/php.ini è il file di config. Il server apache della versione attuale di
MAMP non va oltre la 8.0.1.
Puoi controllare la versione del php del tuo server apache con phpinfo() e puoi anche trovare
il path del file di config utilizzato.


"Drivers not found"
Controlla di aver scritto mysql nel DSN dato che è il nome del driver.
Guarda phpinfo() e controlla che sotto PDO ci sia il driver di mysql.
Cerca extension_dir nel tuo php.ini e assicurati che la cartella sia MAMP/bin/php/php8.0.1/ext.
Assicurati che ci sia extension=php_pdo_sql.dll nel file .ini e che il driver sia effettivamente
presente nella cartella ext.

Il routing non funziona, aggiungere /test o qualunque URI dopo localhost dà errore 404:
Nella root directory del server, tipicamente htdocs, aggiungi un file chiamato .htaccess e
scrivi questo dentro:
	RewriteEngine On
	RewriteRule ^.*$ index.php [END]
	
	
Non mostra gli errori:
ini_set('display_errors', 1); ini_set('display_startup_errors', 1); nel codice
OPPURE vai nel file .ini e mettili manualmente a On
	
	
CSS non funziona:
	il file non viene proprio riconosciuto, non esiste:
		nel file di prima (.htaccess), sopra quelle due righe metti questa:
			RewriteCond %{REQUEST_FILENAME} !-f
		inoltre il path del CSS su nell'attributo href deve partire PER FORZA dalla root del
		server. htdocs per MAMP
	lo stile usato è di un file vecchio o che non esiste più, dov'è?
		nella cache del tuo browser. Fai Ctrl + F5 per eliminare la cache.
		se non funziona fai Ctrl + F5 sulla pagina del CSS da developer tools (Ctrl + Shift + J)


	
Installazione composer

Fai un file chiamato composer.json nella root del tuo progetto e scrivi questo dentro:
{
    "autoload":{
        "classmap": [
            "./"
        ]
    }
}
se ti serve aggiungi
, "exclude-from-classmap": ["/some-direcory/"] dopo classmap

https://getcomposer.org/download/
Scarica l'installer per Windows, te lo mette nel path in automatico quindi il prompt
riconoscerà composer. Vai nella directory di htdocs (usa cd per cambiare directory) o ovunque
sia il tuo progetto e butta un composer install.
Assicurati di aver già fatto il JSON prima di installare

Quando fai nuove classi non farà auto load, ti serve andare nella directory di lavoro col cmd
(htdocs nel caso di mamp) ed eseguire il comando composer dump-autoload.
Va rifatto ogni volta che hai una nuova classe
